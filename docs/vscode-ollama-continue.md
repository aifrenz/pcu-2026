# VSCode에서 로컬 LLM 코드 어시스트 — Ollama + Continue

내 PC 안에서만 동작하는 **로컬 LLM**을 VSCode에 연결해, 인터넷 없이 무료로
코드 작성·설명·자동완성을 사용하는 방법입니다.
**Ollama**(로컬 LLM 실행기) + **Continue**(VSCode 확장 프로그램)을 함께 씁니다.

---

## 1. 소개

보통 AI 코딩 도구(Claude·Codex·Gemini)는 인터넷 너머의 서버에서 동작합니다.
대신 **로컬 LLM**은 AI 모델을 **내 PC에 직접 내려받아 실행**합니다.

| 구분 | 클라우드 AI (Claude 등) | 로컬 LLM (Ollama) |
|---|---|---|
| 동작 위치 | 회사 서버 | **내 PC** |
| 인터넷 | 필요 | 모델 설치 후 **불필요** |
| 비용 | 플랜·사용량 과금 | **무료** |
| 보안 | 코드가 외부로 전송됨 | 코드가 **PC 밖으로 안 나감** |
| 성능 | 매우 강력 | PC 사양에 따라 제한적, 느릴 수 있음 |

> 💡 한 줄 요약: **무료·오프라인·보안**이 장점, **속도·성능**이 단점입니다.
> 민감한 코드를 다루거나 인터넷이 불안정한 환경에서 특히 쓸모가 있습니다.

- **Ollama**: 로컬 LLM을 내려받아 실행해 주는 프로그램
- **Continue**: VSCode 안에서 그 모델과 대화하고 코드를 다루게 해 주는 확장 프로그램

---

## 2. 준비물

| 항목 | 설명 |
|---|---|
| VSCode | 설치 완료 (루트 [README](../README.md)의 "개발 환경 준비" 참고) |
| PC 메모리(RAM) | 모델 크기에 따라 다름 — 아래 표 참고 |
| 디스크 여유 공간 | 모델 1개당 약 **2~9GB** (여러 개면 더 필요) |

### 모델 크기와 PC 사양

| 모델 크기 | 권장 RAM | 적합한 PC |
|---|---|---|
| 1.5B ~ 3B | 8GB 이상 | 사양이 낮은 PC, 자동완성용 |
| 7B | 16GB 이상 | **일반 실습 권장** |
| 14B 이상 | 32GB 이상 / GPU | 고사양 PC |

> ⚠️ PC 사양이 낮으면 큰 모델은 매우 느리거나 멈출 수 있습니다.
> 사양이 부족하면 더 **작은 모델(3B 이하)** 을 사용하세요.

---

## 3. Ollama 설치

1. **다운로드**: [ollama.com](https://ollama.com) 에 접속해 **Windows용** 설치 파일을 받습니다.
2. **설치**: 받은 파일(`OllamaSetup.exe`)을 실행하고 안내대로 설치합니다.
3. **자동 실행**: 설치가 끝나면 Ollama가 백그라운드에서 실행됩니다.
   (작업 표시줄 오른쪽 아래 아이콘으로 확인 가능)
4. **설치 확인**: VSCode에서 터미널(`Terminal → New Terminal`)을 열고 아래를 입력합니다.

   ```powershell
   ollama --version
   ```

   `ollama version is ...` 처럼 버전이 나오면 성공입니다.

> 💡 Ollama는 내 PC 안에 작은 서버(`http://localhost:11434`)를 띄워 둡니다.
> Continue는 이 주소로 모델과 대화합니다. 따로 설정할 필요는 없습니다.

---

## 4. 코딩용 모델 내려받기

모델은 `ollama pull` 명령으로 내려받습니다. **코딩에 좋은 모델**을 받습니다.

```powershell
# 대화·코드 편집용 모델 (7B — 일반 실습 권장)
ollama pull qwen2.5-coder:7b

# 자동완성용 작은 모델 (1.5B — 빠른 응답)
ollama pull qwen2.5-coder:1.5b-base
```

- 모델은 크기가 커서 다운로드에 **몇 분** 걸릴 수 있습니다.
- PC 사양이 낮으면 `qwen2.5-coder:7b` 대신 **`qwen2.5-coder:3b`** 를 받으세요.
- 내려받은 모델 목록은 `ollama list` 로 확인합니다.
- 모델이 잘 동작하는지 바로 시험해 볼 수 있습니다.

  ```powershell
  ollama run qwen2.5-coder:7b
  ```

  질문을 입력해 답이 나오면 정상입니다. (`/bye` 를 입력하면 종료)

> 💡 자동완성용으로 `-base` 가 붙은 모델을 쓰는 이유: 코드 중간을 채우는
> 작업(fill-in-the-middle)에 더 적합하기 때문입니다.

## 5. Continue 확장 프로그램 설치

1. VSCode 왼쪽 **활동 막대**에서 **확장(Extensions)** 아이콘을 클릭합니다.
   (단축키: `Ctrl + Shift + X`)
2. 검색창에 **`Continue`** 를 입력합니다.
3. 게시자(publisher)가 **Continue** 인 항목을 확인한 뒤 **Install** 버튼을 누릅니다.
4. 설치가 끝나면 왼쪽 활동 막대에 **Continue 아이콘**이 생깁니다.

---

## 6. Continue에 Ollama 모델 연결

Continue가 방금 설치한 로컬 모델을 쓰도록 연결합니다.

### 방법 A — 자동 인식 (간단)

1. 왼쪽 활동 막대의 **Continue 아이콘**을 클릭해 패널을 엽니다.
2. 채팅 입력창 아래의 **모델 선택 메뉴**를 엽니다.
3. **Ollama** 를 선택하면, 내려받아 둔 모델들이 목록에 나타납니다.
4. 사용할 모델(예: `qwen2.5-coder:7b`)을 고릅니다.

### 방법 B — 설정 파일로 직접 지정 (자동완성까지 설정)

자동완성 모델은 설정 파일에서 지정하는 것이 확실합니다.

1. Continue 패널 오른쪽 위 **설정(톱니바퀴) 아이콘**을 눌러 설정 파일을 엽니다.
   (파일 위치: `C:\Users\사용자이름\.continue\config.yaml`)
2. `models` 항목에 아래 내용을 넣습니다.

   ```yaml
   models:
     - name: Qwen2.5 Coder 7B
       provider: ollama
       model: qwen2.5-coder:7b
       roles:
         - chat
         - edit
         - apply

     - name: Qwen2.5 Coder 1.5B
       provider: ollama
       model: qwen2.5-coder:1.5b-base
       roles:
         - autocomplete
   ```

3. 파일을 저장하면(`Ctrl + S`) Continue가 설정을 다시 읽습니다.

- `provider: ollama` — 내 PC의 Ollama에 연결한다는 뜻 (주소는 자동)
- `model:` — 4장에서 `ollama pull` 로 받은 모델 이름과 **정확히 같아야** 합니다.
- `roles:` — 그 모델을 어떤 용도로 쓸지 (`chat` 대화, `edit` 코드 수정, `autocomplete` 자동완성)

> ⚠️ Continue 설정 파일 형식은 버전에 따라 조금씩 다를 수 있습니다.
> 형식이 맞지 않으면 패널에 오류가 표시되니, 안내에 따라 수정하세요.

## 7. 사용 방법

### 채팅으로 질문하기
- Continue 패널 입력창에 한국어로 질문합니다.
  - 예) "이 함수가 하는 일을 설명해줘", "이 코드의 버그를 찾아줘"

### 코드에 대해 질문하기
- 편집기에서 코드를 **드래그로 선택**한 뒤 Continue 패널로 질문하면, 그 코드를 기준으로 답합니다.
- 선택 후 `Ctrl + L` 을 누르면 선택한 코드가 바로 Continue 채팅에 들어갑니다.

### 코드 수정 맡기기
- 코드를 선택하고 `Ctrl + I` 를 누른 뒤 "○○로 바꿔줘" 라고 요청하면,
  Continue가 **수정안을 제안**합니다. 검토 후 **수락(Accept)** 하면 반영됩니다.

### 자동완성 (Tab)
- 코드를 입력하는 동안, 자동완성 모델이 **다음 코드를 회색 글씨로 미리 제안**합니다.
- `Tab` 키를 누르면 제안이 입력됩니다.

> 💡 로컬 모델은 클라우드 AI보다 **첫 응답이 느릴 수 있습니다.** 멈춘 것이 아니니
> 잠시 기다리세요. 너무 느리면 더 작은 모델로 바꾸면 빨라집니다.

## 8. 자주 묻는 점 / 주의

| 증상 | 원인 | 해결 |
|---|---|---|
| `'ollama'은(는) ... 인식할 수 없습니다` | Ollama 미설치 / 터미널 미갱신 | Ollama 설치 후 VSCode를 닫았다 다시 열기 |
| Continue에 모델이 안 보임 | 모델을 안 받음 | `ollama pull ...` 로 모델을 받고, `ollama list` 로 확인 |
| 응답이 매우 느림 / 멈춤 | PC 사양 대비 모델이 큼 | 더 작은 모델(3B 이하)로 교체 |
| 응답 품질이 낮음 | 로컬 모델의 한계 | 더 큰 모델 사용 또는 클라우드 AI 병행 |
| 자동완성이 안 됨 | `autocomplete` 역할 모델 미설정 | 6장 방법 B로 자동완성 모델 지정 |
| 디스크 공간 부족 | 모델 파일이 큼 | 안 쓰는 모델 삭제 (`ollama rm 모델이름`) |

> ⚠️ 로컬 LLM은 무료·오프라인이 장점이지만, **성능은 클라우드 AI보다 약합니다.**
> AI가 만든 코드는 로컬·클라우드 상관없이 항상 **사람이 검토**해야 합니다.
